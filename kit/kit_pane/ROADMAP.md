# kit_pane Roadmap

## Mission

Provide reusable pane-shell visuals and authoring overlays for any `core_pane`-driven workspace without taking ownership of pane topology or host policy.

## Immediate Steps

1. stabilize chrome and splitter draw contracts,
2. add explicit authoring overlay primitives (labels/constraints),
3. add compact/dense style profile variants.

## Future Steps

1. add optional header action-slot helpers,
2. add a lightweight validation harness integrated with workspace sandbox,
3. deepen accessibility-focused visual states and focus cues.
